## Standard workflow (preferred — Codex via MCP)

1. Claude works out the change with the user in plain language.
2. Claude sends the plan to Codex for critique (issues/risks only, no code).
3. For large/architectural changes, Claude states the finalized plan and waits for the
   user's "ok". Small changes proceed.
4. Codex implements + runs `npx tsc --noEmit -p tsconfig.app.json` and `npm test`.
5. Claude reads the actual diff and judges matches / diverges (max 2 correction rounds).
6. On "matches": Claude deploys per `AGENTS.md` (frontend build+push and/or Supabase edge
   deploy) and delivers the End-of-work report.
